A. DESKRIPSI SINGKAT
--------------------
Aplikasi web-based untuk mempermudah tim Unit Kendala mengelola data
kendala Indihome/Indibiz dalam jumlah massal. Sumber data utama
adalah Google Sheets yang di-share dengan 3 unit kerja. FilterIN
merupakan layer pintar di atas Google Sheets yang memberikan:

  - Quick Edit Modal (tanpa scroll horizontal panjang)
  - Row lock antar user FilterIN (mencegah tabrakan edit)
  - Audit log (siapa-edit-apa-kapan)
  - Notifikasi data baru hasil sinkronisasi BIMA
  - Dark mode + UI modern
  - Password hashing + CSRF + brute-force protection

Dibangun dengan: Python Flask, MySQL (user/lock/audit),
Google Sheets API (master data), Jinja2 Templates.


B. PERSIAPAN SOFTWARE (PREREQUISITES)
-------------------------------------
Pastikan laptop sudah terinstall:

  1. Python 3.9 atau lebih baru.
     -> Saat install, CENTANG "Add Python to PATH"
     -> Verifikasi: buka CMD, ketik `python --version`

  2. XAMPP (untuk MySQL/MariaDB).
     -> Download dari apachefriends.org
     -> Cukup centang modul Apache + MySQL

  3. Visual Studio Code (editor).
     -> Install extension: "Python" (Microsoft), "Jinja"

  4. Google Chrome / Edge (browser modern).

  5. Git (opsional, untuk clone dari GitHub).


C. CARA INSTALASI PERTAMA KALI (STEP-BY-STEP)
---------------------------------------------

  C.1  COPY / CLONE FOLDER PROJECT
  --------------------------------
  Copy seluruh folder project ini ke laptop (misal ke C:\filterin\).
  Pastikan file & folder berikut ADA:

     backend/
        app_flask.py           <- Kode utama aplikasi
        server.py              <- ASGI wrapper (tidak dipakai di lokal)
        schema.sql             <- Struktur database MySQL
        .env                   <- Variabel environment
        credentials.json       <- Kunci Google Sheet (JANGAN DISEBAR!)
        requirements.txt       <- Daftar library Python
        templates/             <- File HTML
        static/                <- CSS, JS, gambar
     notes.txt                 <- File ini

  C.2  BUKA FOLDER DI VS CODE
  ---------------------------
  File > Open Folder > pilih folder project.

  C.3  BUKA TERMINAL DI VS CODE
  -----------------------------
  Menu: Terminal > New Terminal  (atau Ctrl + `)
  Pastikan terminal berada di ROOT project (bukan di folder backend).

  C.4  BUAT VIRTUAL ENVIRONMENT (REKOMENDASI)
  -------------------------------------------
  Di terminal ketik:

     python -m venv venv

  Lalu aktifkan:

     Windows CMD   :  venv\Scripts\activate
     Windows PowerShell :  venv\Scripts\Activate.ps1
     Mac/Linux     :  source venv/bin/activate

  Kalau berhasil, prompt akan ada tulisan (venv) di depan.

  C.5  INSTALL SEMUA LIBRARY PYTHON
  ---------------------------------
  Di terminal (masih aktif venv):

     pip install -r backend/requirements.txt

  Atau manual satu-satu:

     pip install flask flask-wtf flask-limiter pymysql gspread ^
                 google-auth pandas openpyxl xlrd lxml waitress ^
                 werkzeug python-dotenv requests

  Kalau lambat, pakai mirror:
     pip install -r backend/requirements.txt -i https://pypi.org/simple

  C.6  PASTIKAN FILE credentials.json TERSEDIA
  --------------------------------------------
  File ini berisi KUNCI service account Google.
  JANGAN DIHAPUS dan JANGAN DI-UPLOAD ke GitHub/publik.
  Letaknya harus di:  backend/credentials.json


D. SETTING DATABASE MYSQL
-------------------------

  D.1  NYALAKAN XAMPP
  -------------------
  Buka XAMPP Control Panel, Klik Start di:
     - Apache

  D.2  BUAT DATABASE DAN USER
  ---------------------------
  Buka browser ke: http://localhost/phpmyadmin

  Pilih tab "SQL" di bagian atas, paste perintah ini lalu klik "Go":

        CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

     CREATE USER IF NOT EXISTS 'filterin'@'localhost'
        IDENTIFIED BY 'filterin_pass_2026';

     GRANT ALL PRIVILEGES ON user_db.* TO 'filterin'@'localhost';
     FLUSH PRIVILEGES;

  D.3  IMPORT STRUKTUR TABEL
  --------------------------
  Di phpMyAdmin, klik database "user_db" di sidebar kiri.

     - audit_log       (jejak siapa edit apa)
     - sync_new_rows   (tracking data baru dari sync BIMA)

F. CARA MENJALANKAN APLIKASI
----------------------------

  F.1  CARA 1 — DEVELOPMENT (untuk edit kode)
  -------------------------------------------
  Buka terminal di VS Code (aktifkan venv dulu), lalu:

     cd backend
     python app_flask.py

  Akses di browser:  http://127.0.0.1:5000

  Ctrl + C untuk stop.
  Hot-reload: setiap ubah kode Python/template, server auto-restart.

  F.2  CARA 2 — PRODUCTION (untuk dipakai banyak orang di kantor)
  ---------------------------------------------------------------
  Pakai waitress (WSGI server, lebih stabil dari flask dev server):

     cd backend
     waitress-serve --host=0.0.0.0 --port=5000 app_flask:flask_app

  Akses dari laptop server   :  http://127.0.0.1:5000
  Akses dari laptop lain di  :  http://192.168.x.xx:5000
  jaringan yang sama            (ganti IP dengan IP laptop server)

  Untuk tahu IP laptop:  buka CMD, ketik  ipconfig
  Cari "IPv4 Address" di bagian Wireless/Ethernet Adapter.

  F.3  CARA 3 — PAKAI FILE BAT (PALING PRAKTIS)
  ---------------------------------------------
  Buat file  start_filterin.bat  di root project, isinya:

     @echo off
     TITLE FilterIN Server
     echo Starting FilterIN on port 5000...
     cd backend
     call ..\venv\Scripts\activate
     waitress-serve --host=0.0.0.0 --port=5000 app_flask:flask_app
     pause

  Klik 2x file tersebut → CMD terbuka, biarkan minimize.
  Akses di browser sesuai F.2.

H. FITUR-FITUR BARU (versi 2.0)
-------------------------------

  H.1  QUICK EDIT MODAL (pengganti edit inline)
       Di halaman Sheet Kendala Master, klik tombol pensil ✏️
       di sebelah kanan tiap baris. Modal akan muncul berisi:
          - Context read-only: ORDER_ID, WONUM, STO, DATEL,
            SUB ERROR CODE, ENGINEER MEMO, STATUS, ORDER DATE
          - Form editable: ACTUAL KENDALA, FEEDBACK ASO,
            TGL FEEDBACK (otomatis), NOTES ASO, IS_ACTIVE_KENDALA
       Tidak perlu scroll kanan-kiri lagi!
       Shortcut: Ctrl+S = Simpan, Esc = Batal.

  H.2  PILIH KOLOM & MODE CEPAT
       Klik "Pilih Kolom" untuk centang/uncentang kolom yang mau
       ditampilkan. Pilihan disimpan di browser (localStorage),
       beda user bisa punya tampilan sendiri.
       Klik "Mode Cepat" untuk langsung tampilkan hanya kolom
       kerja (ORDER_ID, WONUM, STO, SUBERROR, MEMO, ACTUAL,
       FEEDBACK, NOTES, IS_ACTIVE).

  H.3  NOTIFIKASI DATA BARU
       Setelah klik "Sinkronkan Data BIMA":
          - Banner oranye muncul: "{N} data baru baru saja
            disinkronkan!"
          - Tombol "Lihat Data Baru" untuk filter cepat.
          - Baris baru ditandai badge "NEW" + garis oranye
            di sebelah kiri.
          - Tombol "Tandai Sudah Dibaca" untuk dismiss.

  H.4  ROW LOCK (anti tabrakan antar user FilterIN)
       Saat user A buka Quick Edit Modal row X, row tersebut
       ter-lock selama 5 menit. User B yang buka row yang sama
       akan lihat warning: "Row sedang diedit oleh Andi
       (sejak 14:23)" dan tombol Save dinonaktifkan.
       Lock otomatis release setelah save / cancel / 5 menit.

       CATATAN: Lock ini hanya berlaku ANTAR USER FILTERIN.
       Kalau unit kerja lain edit sheet langsung via
       docs.google.com, kami tidak bisa mengunci mereka.

  H.5  AUDIT LOG (khusus admin)
       Menu sidebar "Audit Log" (hanya muncul untuk admin).
       Berisi tabel: kapan, siapa, aksi apa, row mana, kolom apa,
       nilai lama → nilai baru. Bisa filter per user / action.

  H.6  DARK MODE
       Tombol "Dark Mode" di bagian bawah sidebar.
       Pilihan disimpan di browser per user.

  H.7  UI MODERN
       - Login page: toggle show/hide password, gradient biru Telkom
       - Flash toast auto-close 5 detik, bisa di-close manual
       - Loading overlay global saat upload / sync
       - Konfirmasi dialog sebelum aksi destruktif (Sync, Move UNSC)
       - Badge role (ADMIN/OPERATOR) di header

J. TROUBLESHOOTING
------------------

  J.1  ERROR "Can't connect to MySQL server"
       -> Cek XAMPP: MySQL sudah di-Start?
       -> Cek port 3306 tidak bentrok dengan aplikasi lain.
       -> Cek user & password di .env sama dengan yang di phpMyAdmin.

  J.2  ERROR "FileNotFoundError: credentials.json"
       -> File harus di folder backend/credentials.json
       -> Bukan di root, dan bukan di folder lain.

  J.3  ERROR "Spreadsheet not found" / "PermissionDenied"
       -> Cek internet.
       -> Cek service account sudah di-share ke sheet (lihat I).
       -> Cek SPREADSHEET_* IDs di .env benar.

  J.4  UPLOAD FILE GAGAL
       -> Format harus .xlsx, .xls, atau .csv.
       -> Cek kolom wajib di file Excel:
             SC Order No/Track ID/CSRM No, CRM Order Type, Status
          (untuk Filter BIMA)

  J.5  LOGIN SELALU GAGAL PADAHAL PASSWORD BENAR
       -> Cek 5 menit terakhir gagal > 5x? Sistem auto-block.
          Tunggu 5 menit atau hapus record di tabel login_attempts:
             DELETE FROM login_attempts WHERE username='USERNAME_ANDA';

  J.6  TOMBOL "SIMPAN PERUBAHAN" DISABLED DI MODAL
       -> Berarti row sedang di-lock oleh user lain.
       -> Tunggu 5 menit atau minta user lain close modalnya.
       -> Admin bisa paksa clear lock via phpMyAdmin:
             DELETE FROM edit_locks WHERE row_key='ORDER_ID_ANDA';

  J.7  TAMPILAN WEB RUSAK / MENU HILANG
       -> Ctrl + F5 (hard refresh browser, clear cache).
       -> Clear localStorage: F12 > Application > Local Storage > Clear.

  J.8  WATERFALL / POLLING TERLALU BANYAK (RATE LIMIT GOOGLE)
       -> Naikkan SHEET_CACHE_TTL di .env (default 30 detik).
          Contoh: SHEET_CACHE_TTL=60 untuk cache 1 menit.

  J.9  INGIN RESET SEMUA LOCK
       Di phpMyAdmin > user_db > tab SQL:
          DELETE FROM edit_locks;

  J.10 PASSWORD USER LUPA
       Bikin ulang via phpMyAdmin:
          UPDATE users SET password='BARU123' WHERE username='XXX';
       Password akan ter-hash saat login pertama.

L. CATATAN ARSITEKTUR (PENTING UNTUK DEVELOPER)
-----------------------------------------------

  L.1  Source of Truth tetap Google Sheets
       FilterIN TIDAK memindah data ke MySQL. Google Sheet tetap
       jadi database utama karena di-share dengan 3 unit kerja.
       MySQL HANYA untuk: users, edit_locks, audit_log,
       sync_new_rows, login_attempts.

  L.2  Hybrid Architecture
       - Unit Kendala (Anda): edit via FilterIN web (cepat + safe)
       - Unit lain (TSEL, TATI): edit Google Sheet langsung
       - Zero disruption untuk unit lain.
       - Saat Anda save di FilterIN, data langsung ke Google Sheet.
       - Saat unit lain edit Sheet, FilterIN lihat perubahan setelah
         cache refresh (30 detik).

  L.3  Kolom "kepemilikan" (kesepakatan tim)
       - A-K (WONUM, ORDER_ID, ...) : otomatis dari sync BIMA
       - FEEDBACK ASO, ACTUAL KENDALA, NOTES ASO : Unit Kendala
       - FEEDBACK TSEL, NOTES TSEL               : Unit TSEL
       - DONE TATI, NOTES TL, UPDATE TATI        : Unit TATI
       Kalau dua unit edit kolom yang SAMA di waktu yang SAMA,
       Google Sheet native yang menang (last write wins).
